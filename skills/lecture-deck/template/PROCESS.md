# 강의안 제작 프로세스 — PPTX에서 HTML로

> 한 줄 요약: **마크다운(`slides.md`) 한 파일만 쓰면, 화면용 발표·인쇄용 배포본·웹 배포가 모두 자동으로 나옵니다.**
> PowerPoint처럼 "도형을 직접 배치"하지 않고, "내용만 글로 적으면" 디자인은 엔진(`deck.html`)이 입힙니다.

---

## 0. 왜 이 방식인가 (PPTX와의 차이)

| | PPTX 방식 | 이 HTML 방식 |
|---|---|---|
| 원본 | `.pptx` 바이너리 | `slides.md` 텍스트(누구나 편집·버전관리) |
| 디자인 | 슬라이드마다 수동 배치 | 레이아웃 = 한 줄 지시어로 자동 |
| 수정 | 60장 폰트 바꾸려면 60번 | CSS 변수 1줄로 전체 변경 |
| 배포 | 파일 전송, 버전 꼬임 | URL 하나(`#s27`로 특정 장 링크) |
| 인쇄 | 그대로 | 같은 원본에서 **배포용 PDF 자동 생성** |
| AI 협업 | 어려움 | 텍스트라 **AI가 초안·수정 직접 작성** |

참고한 3개 사이트(allrounder.im, be-ax-workflow, obsidianmrchoi)가 전부 이 구조입니다 — **마크다운 → 슬라이드형 HTML**.

---

## 1. 폴더 구조

```
HTML강의플러그인/
├── deck.html        ← 엔진. 건드리지 않습니다(테마 색만 가끔 조정).
├── slides.md        ← ⭐ 여기에만 강의 내용을 씁니다.
├── build.py         ← 배포용 단일 파일 만드는 스크립트.
├── assets/          ← 이미지·사진·로고를 여기에 모읍니다.
├── _snippets/       ← 레이아웃별 복붙용 마크다운 예시.
├── dist/            ← build.py 결과물(배포본). 자동 생성.
└── PROCESS.md       ← 이 문서.
```

새 강의를 만들 땐 이 폴더를 통째로 복사해서 `slides.md`만 새로 쓰면 됩니다. (예: `강의_PR마스터/`, `강의_AIEO/` …)

## 2. 전체 제작 흐름 (5단계)

```
① 설계        ② 작성            ③ 미리보기         ④ 이미지         ⑤ 배포
PPT/PDF/머릿속 → slides.md 작성 → 로컬서버로 확인 → assets/ 채우기 → 웹/PDF/단일파일
   (아웃라인)     (마크다운)       (실시간 반영)      (캡처·최적화)     (build.py)
```

### ① 설계 — 마크다운 아웃라인 먼저 뽑기

PPT를 바로 옮기지 말고, **글의 뼈대(아웃라인)부터** 마크다운으로 만듭니다. 기존 PPTX/PDF가 있으면 AI에게 이렇게 시킵니다:

> "첨부한 PDF(강의안)를 슬라이드 단위로 분해해서, 아래 규칙의 마크다운 아웃라인으로 만들어줘.
> · 슬라이드 구분은 `---`
> · 각 장은 `## 제목` + 핵심 3~5개 불릿
> · 표지/섹션 구분 장은 `<!-- title -->` `<!-- section -->` 지시어
> · 사진이 들어갈 자리는 `![설명](assets/파일명.jpg)` 자리표시자로 남겨줘"

→ 이렇게 하면 158장 PPT도 30분이면 마크다운 초안이 나옵니다. (이번 `slides.md`가 그 예시입니다.)

### ② 작성 — 두 가지 방법

**방법 A — 에디터(`editor.html`) · 권장, 마크다운 몰라도 됨**
`http://localhost:8000/editor.html` 접속 → obsidian 데크처럼 편집합니다.
- 왼쪽 **필름스트립**: 슬라이드 클릭 선택 / `＋`추가 / `⧉`복제 / `🗑`삭제 / `↑↓`순서이동
- 가운데 **편집창** + 서식 버튼(제목·굵게·목록·인용·통계·2단·노트) → 문법 몰라도 클릭으로 삽입
- **레이아웃 드롭다운**: 표지·섹션·가운데·다크·배경사진·2단 한 번에 적용
- **🖼 이미지**: 파일 선택 → `assets/`에 자동 저장 + 슬라이드에 삽입
- 오른쪽 **실시간 미리보기**(발표 화면과 100% 동일)
- **📁 폴더 연결**(크롬·엣지): 한 번 누르면 `slides.md`에 **자동 저장**. 이후 `deck.html`과 즉시 동기화.
- **▶ 발표 / ▦ 개요 / 🖨 인쇄**도 에디터 안에서 바로.

> 폴더 연결은 크롬·엣지에서 `localhost`로 열 때만 동작합니다(브라우저 보안). 사파리/파이어폭스는 '저장' 시 `slides.md`를 내려받아 폴더에 덮어쓰면 됩니다.

**방법 B — `slides.md` 직접 편집**
텍스트 에디터로 마크다운(아래 4장)을 직접 씁니다. AI에게 초안·수정을 시키기에 가장 좋습니다. 저장 후 새로고침하면 반영됩니다.

> 두 방법은 같은 `slides.md`를 공유합니다. 에디터로 틀을 잡고, 대량 수정은 AI로 — 자유롭게 오가며 쓰세요.

### ③ 미리보기 — 로컬 서버로 실시간 확인

`slides.md`를 저장하고 브라우저에서 새로고침하면 바로 반영됩니다. 폴더에서 터미널을 열고:

```bash
python3 -m http.server 8000
```

브라우저에서 `http://localhost:8000/deck.html` 접속. (맥은 파이썬이 기본 설치돼 있습니다.)

> 터미널이 부담스러우면: `slides.md`를 저장한 뒤 `python3 build.py`를 한 번 돌리고 `dist/index.html`을 더블클릭해도 됩니다.

### ④ 이미지 처리 (아래 5장 상세)

`assets/`에 사진을 모으고 `![](assets/...)` 로 연결합니다.

### ⑤ 배포 (아래 6장 상세)

- **웹**: `slides.md` + `deck.html` + `assets/`를 Netlify/Vercel에 올리면 끝.
- **단일 파일/오프라인**: `python3 build.py` → `dist/index.html` (더블클릭으로 열림).
- **인쇄 배포용 PDF**: 브라우저에서 `P` → 'PDF로 저장'.

## 3. 발표할 때 (조작)

| 동작 | 기능 |
|---|---|
| **화면 클릭** | 다음 슬라이드 |
| **우클릭** | 이전 슬라이드 (우클릭 메뉴 안 뜸) |
| **프레젠터 리모컨** | PageUp/PageDown → 이전/다음 (그대로 작동) |
| `←` `→` (또는 스페이스) | 이전/다음 |
| `F` | 전체화면 |
| `O` | 한눈에 보기(전체 슬라이드 격자) |
| `S` | 발표자 노트 켜기/끄기 |
| `P` | 인쇄(배포용 PDF) |
| `.` | 화면 깜깜이(주의 집중) |
| `?` | 도움말 표시/숨김 |
| `Home` `End` | 처음/끝으로 |

> 슬라이드 안 **링크는 클릭해도 넘어가지 않고** 링크가 열립니다. 단축키는 물리 키(`e.code`) 기반이라 한/영 입력 상태와 무관하게 작동합니다.

URL `…/deck.html#s27` 처럼 특정 장으로 바로 링크할 수 있습니다(카톡 공유·이어보기에 유용).

> 슬라이드를 넘기면 좌우 슬라이드+페이드 전환이 들어가고, 본문 요소가 순차적으로 등장(스태거)합니다. 이 모션은 발표 화면(데크·발표모드)에서만 작동하고, 에디터 미리보기·인쇄에는 영향이 없습니다. OS의 '동작 줄이기(prefers-reduced-motion)' 설정을 켜면 자동으로 전환이 꺼집니다.

## 4. 마크다운 문법 레퍼런스

### 슬라이드 구분
빈 줄에 `---` 한 줄. 그 사이가 한 장입니다.

### 레이아웃 지시어 (장 맨 윗줄, 선택)
```
<!-- title -->            표지(크게, 가운데)
<!-- section -->          파트 구분(강조색 배경)
<!-- center -->           내용 가운데 정렬
<!-- dark -->             어두운 배경
<!-- bg:assets/a.jpg -->  사진 풀배경 + 글자 흰색
<!-- accent:#e11d48 -->   이 장만 강조색 변경
```
여러 개 조합 가능: `<!-- section accent:#e11d48 -->`

### 글
```
^ 라벨            (제목 위 작은 청록 라벨 = kicker. 파트·구획 표시에 좋음)
## 제목            (장 제목)
### 소제목
**굵게**  *강조(포인트색)*  ~~취소선~~  `코드`
- 불릿            1. 번호목록
[링크텍스트](https://…)
```
> `^ 라벨` 은 obsidian 스타일의 시그니처 요소입니다 — 청록 글로우 선 + 모노 대문자 라벨.

### 이미지 자리 · 출처
```
@img 설명          (점선 이미지 자리 → 에디터 🖼 버튼으로 실제 사진 교체)
@src 출처 텍스트    (슬라이드 하단 작은 회색 '출처' 표기. 여러 줄 가능)
```
> **원칙: 통계·수치가 들어간 슬라이드에는 반드시 `@src` 로 출처를 답니다.** 출처가 불확실하면 `@src *원본 확인 필요* — 설명` 으로 표시해 두고 확정 후 교체합니다.

### 인용 (출처는 마지막 줄에 `— `)
```
> 적은 게 강하다.
> — 어느 카피라이터
```

### 표
```
| 구분 | 내용 |
|---|---|
| 언론 PR | 인터뷰, 기고 |
```

### 카드 그리드 — 마우스 호버 효과
```
::: cards
### 라벨
**제목**
설명
+++
### 라벨
**제목**
설명
:::
```
> 마우스 올리면 테두리·떠오름. 마우스 따라오는 글로우는 자동.

### 2단 컬럼 — `+++` 로 좌/우 분할
```
## 비교

왼쪽 내용
+++
오른쪽 내용
```

### 발표자 노트 — `???` 아래 (화면엔 안 보이고 `S`로만 봄)
```
## 슬라이드 제목
본문...
???
여기는 나만 보는 멘트. "여기서 손들기 유도" 같은 진행 메모.
```

> 전체 예시는 `slides.md`와 `_snippets/` 폴더를 보세요.

## 5. 이미지 처리 (가장 자주 묻는 부분)

### 원칙
1. 모든 이미지는 **`assets/` 폴더 한 곳**에 모읍니다.
2. 파일명은 `날짜-내용.jpg` 식으로(예: `2026-trend-aisub.jpg`). 한글·공백 피하기.
3. 마크다운에서 `![설명](assets/파일명.jpg "캡션")` 으로 연결. 캡션은 생략 가능.

### 기존 PPTX/PDF에서 이미지 빼내기
PDF에서 장면을 그림으로 추출(poppler 설치 시):
```bash
# (1) 슬라이드 전체를 페이지 단위 PNG로 — 그대로 쓰거나 크롭용
pdftoppm -png -r 150 "원본.pdf" assets/page

# (2) PDF 안에 박힌 원본 사진만 추출
pdfimages -png "원본.pdf" assets/img
```
또는 PPT에서 슬라이드를 '그림으로 저장' / 캡처해서 `assets/`에 넣습니다.

### 최적화 (발표가 버벅이지 않게)
- 가로폭 **1600px** 정도면 충분합니다(프로젝터·웹 모두).
- 사진은 `.jpg`, 로고·도식은 `.png`, 가능하면 `.webp`.
```bash
# ImageMagick 예시: assets의 큰 이미지를 일괄 축소
magick mogrify -resize 1600x -quality 82 assets/*.jpg
```

### 배치 요령
- **풀배경 사진** 한 장으로 임팩트: `<!-- bg:assets/crisis.jpg -->` + 제목 한 줄.
- **사진 + 설명** 나란히: `+++` 2단으로 왼쪽 사진, 오른쪽 텍스트.
- 사진은 자동으로 비율 유지하며 슬라이드 안에 맞춰집니다(잘림 없음).

> ⚠️ 단일 파일(`dist/index.html`)로 배포해도 **이미지는 파일에 내장되지 않습니다.** 반드시 `assets/` 폴더를 함께 옮기세요. (build.py가 자동 복사합니다.)

## 6. 배포 — 3가지 경로

### A. 배포본 PDF (수강생 나눠주기)
**가장 쉬운 방법 — 더블클릭**: `PDF로 내보내기.command` 실행 → `dist/강의안.pdf` 생성(자동으로 열림).
- 페이지가 **16:9 슬라이드 크기**로 떨어져 한 장당 한 페이지, 여백 없이 꽉 찹니다.
- 넣어 둔 이미지(assets/)도 그대로 들어갑니다.

**수동 방법**(어느 브라우저든): 데크를 열고 `P`(또는 Cmd/Ctrl+P) → 대상 **PDF로 저장** → **배경 그래픽 켜기** 체크 → 저장.

> 마크다운 원고도 함께 보관하려면 `slides.md` 를 그대로 두거나 `dist/` 에 사본을 둡니다 — PDF·HTML·마크다운이 한 세트입니다.

### B. 단일 파일 (USB·이메일·오프라인 강의장)
```bash
python3 build.py
# → dist/index.html  (더블클릭으로 바로 열림. assets/도 함께 생성)
```
인터넷 없는 강의장에서 가장 안전합니다.

### C. 웹 게시 (URL 공유 · 사용자님이 이미 쓰는 방식)
- **Netlify**: netlify.com → 폴더(`deck.html`,`slides.md`,`assets/`)를 드래그&드롭 → 즉시 URL.
- **Vercel**: 같은 폴더를 올리면 됩니다.
- 깃허브에 올려두면 수정→자동 재배포도 가능.

- [ ] 강의 전: `build.py`로 단일 파일 백업 1부 + 인쇄 PDF 1부

## 8. 테마(색·폰트) — 다크 'Obsidian' 스타일 · 3색 원칙

기본 테마는 obsidianmrchoi 데크와 같은 **다크 차콜 + 흰색 위계 + 블루 강조**입니다.
`slides.css` 의 `:root` 에서 칠하는 색 **3개**만 정합니다.

```css
--bg:#1f2228;     /* ① 배경(차콜) */
--fg:#ffffff;     /* ② 글자(흰색) — 위계는 투명도(--t72/--t50/--t30)로 */
--accent:#3b82f6; /* ③ 강조(블루) — 기관 브랜드 색 1개로 교체 */
--accent2:#22d3ee;/*   시그니처 청록(라벨선·불릿). 1색만 쓰려면 var(--accent) 로 */
```

- 보조 글자·구분선은 **흰색의 투명도 단계**로 표현합니다(새 색 아님).
- **폰트**: 제목·라벨·숫자 = JetBrains Mono, 본문 = Pretendard (CDN 자동 로드. 오프라인이면 시스템 폰트로 대체).
- 한 장만 강조색을 바꾸려면 `<!-- accent:#색 -->`.

**`--accent` 만 바꿔 쓰는 톤 예시:**

| 톤 | --accent |
|---|---|
