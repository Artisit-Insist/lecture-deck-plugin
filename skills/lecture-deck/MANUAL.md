# 강의안 만들기 — 사용 매뉴얼

> Claude Code에게 말로 시키면, 정해진 프로세스로 **HTML 강의안 한 세트**(발표용 deck.html · PDF · 강사 노트 · 원고)를 만들어 줍니다.
> 이 프로세스는 `lecture-deck` 스킬로 박혀 있어, 어느 폴더에서든 동일하게 작동합니다.

---

## 1. 새 강의안 만들기 — 이렇게 말하세요

> **"이 개요로 강의안 만들어줘"** (+ 주제·목차·시간·대상)
> 또는 PDF/문서를 주며 **"이 자료로 강의 슬라이드 만들어줘"**

개요는 이 정도면 충분합니다 (아는 만큼만):
```
주제:        (예: AI로 사내 커뮤니케이션 잘하기)
시간/분량:    (예: 2시간 / 약 30장)
대상:        (예: 중간관리자)
목차:        (큰 흐름 3~6개)
넣을 사례/포인트: (있으면)
톤:          (사례 중심·실습 위주 / 격식 / 가볍게)
```

→ Claude가 새 폴더를 만들고, 위 개요를 살 붙여 **slides.md**로 작성한 뒤 **deck.html**까지 빌드해 줍니다.

## 2. 보기 — deck.html 더블클릭

`deck.html`을 **더블클릭**하면 끝(서버·인터넷 불필요). 모든 내용이 파일 안에 들어있는 자립형입니다.

**넘기기 — 마우스만으로도 OK:**
- **화면 클릭 = 다음**, **우클릭 = 이전** (발표 때 가장 편함)
- **프레젠터 리모컨**(무선 포인터) 그대로 작동
- 키보드 `←` `→` 도 가능

기타: `F` 전체화면 · `S` 강사 노트 · `O` 한눈에 보기 · `P` 인쇄(PDF) · `?` 도움말

## 3. 고치기 — Claude에게 말로

> "3번 슬라이드 사례 바꿔줘" · "M2에 슬라이드 한 장 추가" · "표지 색 바꿔줘" · "이 수치 출처 추가"

Claude가 `slides.md`를 고치고 **자동으로 deck.html을 다시 빌드**합니다. 그다음 deck.html을 새로고침/다시 열면 반영돼 있습니다.

## 4. 한 세트로 나오는 산출물

| 파일 | 용도 |
|---|---|
| `deck.html` | ⭐ 발표 (더블클릭) |
| `dist/강의안.pdf` | 인쇄·배포 (16:9) |
| `강사노트.md` | 강사용 상세 노트 (배경·멘트·예상질문) |
| `dist/..._읽기용.md` | 수강생용 줄글 문서 |
| `slides.md` | 내용 원본 (Claude가 편집) |

## 5. 자료 조사가 들어가는 부분

- **통계·수치 슬라이드에는 항상 출처**가 붙습니다. 출처가 불확실하면 `원본 확인 필요`로 표시되니, 강의 전 그 부분만 확인하세요.
- **강사 노트**는 슬라이드에 못 담은 배경을 웹 리서치로 보강해 작성됩니다(🎯 목적 · 🗣 멘트 · 📚 배경 · ❓ 예상질문 · 💡 팁).

## 6. 이미지

이미지는 **점선 자리(`@img`)**로 표시만 됩니다. 실제 사진은:
- 에디터(`editor.html`)에서 `🖼` 버튼으로 넣거나,
- `assets/` 폴더에 넣고 Claude에게 "○번 슬라이드에 이 사진 넣어줘" 라고 하면 됩니다.
