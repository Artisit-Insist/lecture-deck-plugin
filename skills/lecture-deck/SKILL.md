# 강의안(HTML 데크) 제작 스킬

황상현 코치의 표준 강의안 제작 프로세스. **다크 Obsidian 스타일 · 자립형 deck.html · 통계 출처 표기 · 강사 노트**를 한 세트로 만든다.

## 0. 준비 — 새 강의안이면 템플릿 복사

이 스킬 폴더의 `template/` 가 깨끗한 툴킷이다. 새 강의안은:
1. 사용자에게 저장 위치/강의명을 확인(기본: 현재 작업폴더 아래 `강의명/`).
2. `cp -R "<이 스킬 경로>/template/" "<대상폴더>"` 로 복사.
3. 이후 그 폴더 안에서 작업한다.

기존 강의안 수정이면 해당 폴더에서 바로 작업.

**기존 PPTX/PDF가 있으면** — 대상 폴더에서 `python3 import_deck.py 원본.pdf`(또는 `.pptx`) 로 `slides.md` 초안 + `assets/` 이미지를 뽑은 뒤, 그 초안을 2장 스타일 규칙대로 다듬는다(제목·불릿 정리, 섹션 구분, @img 배치, 통계 @src 표기). 초안은 거칠다 — 반드시 손본다.

> 스킬 경로는 보통 `~/.claude/skills/lecture-deck`. 모르면 `find ~/.claude -maxdepth 3 -name lecture-deck -type d` 로 찾는다.

## 1. 개요 받기 (없으면 묻는다)

사용자에게 **주제·시간/분량·대상·목차·톤**을 묻는다(아는 만큼만). PDF/문서/기존 자료를 주면 그걸 출처로 쓴다. "알아서"면 합리적으로 구성하되 핵심은 확인.

## 2. slides.md 작성 — ★스타일 규칙(반드시 준수)

`slides.md` 한 파일이 모든 산출물의 원천. 문법:

- **슬라이드 구분**: 빈 줄 / 단독 `---` / 빈 줄.
- **표지(첫 슬라이드)**: 첫 줄 `<!-- title -->`, 그 아래 `^ 라벨`, `# 제목`, 부제, 강사·소속·날짜.
- **모듈/파트 구분**: `<!-- section -->` + `^ PART/MODULE N · 시간` + `# 제목` + 부제. 바로 다음에 학습목표 슬라이드 권장.
- **각 슬라이드**: `^ 라벨`(kicker, 선택) → `## 제목` → 내용. **한 장에 핵심 하나, 불릿 3~5개.**
- 강조: `**키워드**`(흰 굵게), `*강조*`(포인트색), 한 문장 임팩트/예시는 `> 인용`.
- 복사용 프롬프트·템플릿·코드는 ``` 코드펜스 ```. 표는 `| a | b |`. 2단은 빈 줄 `+++` 빈 줄. 큰 숫자는 `:: 90%`.
- **카드 그리드(호버 인터랙션)**: `::: cards` 줄 … 카드 사이 `+++` … 닫는 `:::`. 카드 안은 `### 라벨` + `**제목**` + 설명. 마우스 올리면 테두리 강조·살짝 떠오름. (마우스 따라오는 옅은 원 글로우는 모든 슬라이드에 자동 — 별도 작성 불필요)
- **막대 차트**: `::: bars` … `라벨 | 값` 줄들 … `:::`. 값의 숫자로 막대 길이 자동(`모바일 | 68%`). 비중·비교 통계에.
- **흐름도/파이프라인**: `::: flow` … `A → B → C` … `:::`. 단계·프로세스를 박스+화살표로.
- **이미지 자리**: `@img 설명` → 점선 박스. 이미지는 사용자가 직접 넣는다(생성하지 말 것).
- **출처(★중요)**: `@src 출처` → 슬라이드 하단. **수치·통계·날짜·제품사양·업데이트가 든 슬라이드에는 반드시 @src.** 출처가 불확실하면 절대 지어내지 말고 `@src *원본 확인 필요* — 설명`.
- raw HTML 금지. 다크 테마 전제(배경/글자/강조 3색).
- 전체 문법·레이아웃 예시는 `template/slides.md` 와 `_snippets/레이아웃모음.md`, 상세는 `PROCESS.md`.

## 3. 큰 데크는 병렬 생성

여러 모듈(예: 5장 이상 × 여러 파트)이면 **모듈당 에이전트 1개로 병렬 작성**(general-purpose). 각 에이전트는 자기 모듈 슬라이드를 위 문법으로 `/tmp/<deck>_M{n}.md` 에 쓰고, 끝나면 합쳐서 `slides.md` 에 `---` 로 이어붙인다. 출처 허용 목록(부록/자료의 출처)을 각 에이전트에 명시하고 "없는 수치 금지"를 강조.

## 4. 강사 노트 (요청 시 / 기본 동행)

사용자가 강사 노트를 원하면(또는 표준으로) **모듈당 에이전트 1개 병렬**로 `강사노트.md` 작성:
- 형식: 슬라이드별 `### 제목` + 🎯핵심목적 · 🗣이렇게 말하세요 · 📚더 깊은 배경(WebSearch로 보강) · ❓예상 질문&답 · 💡진행 팁.
- 모듈 맨 앞 `## MODULE N · 제목 — 강사 노트` + 운영 포인트.
- 비개발자가 이해할 비유로. **정확성 최우선 — 불확실하면 '확인 필요', 날조 금지.**
- 합친 뒤 **`python3 inject_notes.py`** 로 각 슬라이드에 `???` 발표자 노트로 주입(모듈별 위치정렬/유사도 매칭, 재실행 안전).

## 5. 빌드 — 자립형 deck.html (★매번)

내용을 바꾼 뒤에는 **항상** `python3 build.py` 를 실행한다.
- `_deck_template.html` + slides.css + engine.js + slides.md 를 합쳐 **자립형 `deck.html`**(더블클릭용)과 `dist/index.html`(사본) 생성.
- **deck.html 을 직접 편집하지 말 것** — 빌드가 덮어쓴다. 항상 slides.md → build.py.

## 6. 산출물

- **PDF**: `PDF로 내보내기.command`(더블클릭) 또는 헤드리스 Chrome로 file:// 인쇄 → 16:9 페이지. (참고: `@page{size:13.333in 7.5in}`)
- **읽기용 줄글 문서**: `python3 make_handout.py` → `dist/<제목>_읽기용.md`(노트 제외).
- 발표: `deck.html` 더블클릭. **넘기기 — 마우스 클릭=다음 · 우클릭=이전 · 프레젠터 리모컨(PageUp/Down) · 키보드 `←→`.** 그 외 `F`전체화면 `O`한눈에 `S`노트 `P`인쇄 `.`깜깜이 `?`도움말. (슬라이드 안 링크 클릭은 그대로, 넘김 안 됨)

## 7. 검증 (생략 금지)

- 빌드 후 슬라이드 수 / 파싱 오류 확인. 가능하면 헤드리스 Chrome로 `file://.../deck.html` 열어 렌더 확인(boot 숨김, 슬라이드 수, 표지 정상).
- **`python3 lint.py` 실행** — 통계 슬라이드 @src 누락 · 내용 넘침(불릿 과다·긴 줄) · 깨진 이미지 · 제목 누락을 자동 점검. 경고를 보고 정리한다.

## 핵심 규칙 (요약)
1. 내용은 `slides.md` 한 파일 → 고치면 **반드시 `python3 build.py`**.
2. `deck.html` 은 자립형 빌드 산출물(더블클릭). 직접 편집 금지.
3. **통계엔 반드시 출처(@src)**, 모르면 '원본 확인 필요' — 날조 금지.
4. 이미지는 `@img` 자리만, 실제 사진은 사용자가 삽입.
5. 다크 Obsidian 스타일 · 3색 · 한 장에 핵심 하나.
6. 단축키는 `e.code`(물리 키) 기반이라 한/영 무관. 넘기기는 **마우스 클릭=다음 · 우클릭=이전 · 리모컨(PageUp/Down) · 키보드** 모두 지원(템플릿에 반영됨).
